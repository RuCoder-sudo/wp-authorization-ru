== Description ==

WP Authorization RU — плагин для авторизации пользователей WordPress через российские OAuth-провайдеры. Версия 1.0.1.

**Шесть провайдеров в одном плагине:**

* Яндекс ID — вход через аккаунт Яндекс.
* Mail.ru — вход через аккаунт Mail.ru.
* ВКонтакте — вход через аккаунт VK (OAuth 2.0, email возвращается в ответе токена).
* Rambler — вход через Rambler ID (OpenID Connect / OAuth 2.0).
* MAX — вход через мессенджер MAX (Mini App, HMAC-SHA256 валидация initData).
* Госуслуги — вход через ЕСИА (OAuth 2.0; требует УКЭП для продуктива, поддерживает тестовую среду).

**Автообновление с GitHub:**

Плагин автоматически проверяет новые версии на GitHub (репозиторий RuCoder-sudo/wp-authorization-ru) каждые 12 часов. При выходе новой версии в панели управления WordPress появляется стандартное уведомление — обновление устанавливается одним кликом.

**Где появляются кнопки входа:**

* Стандартная страница входа WordPress (wp-login.php)
* Стандартная страница регистрации WordPress
* Страница «Мой аккаунт» WooCommerce (форма входа)
* Страница «Мой аккаунт» WooCommerce (форма регистрации)
* Страница оформления заказа WooCommerce — баннер «Войдите перед оформлением»

**Что происходит при входе:**

* Пользователь нажимает кнопку нужного провайдера.
* Перенаправляется на сервис для подтверждения.
* После подтверждения возвращается на сайт уже авторизованным.
* При первом входе автоматически создаётся аккаунт WordPress (роль: Подписчик).
* При повторном входе — распознавание по OAuth ID или email.
* Имя и фамилия из профиля провайдера сохраняются в профиле WordPress.

**Безопасность:**

* CSRF-защита через state-параметр (WordPress nonce + transient, срок действия 10 минут).
* Callback через WordPress REST API — чистые URL без query-параметров.
* Все входные данные проходят санитизацию через функции WordPress.
* Редирект только на страницы того же сайта.
* MAX: HMAC-SHA256 подпись с проверкой auth_date (не старше 1 часа).

**Совместимость:** WordPress 5.8 – 6.7 · PHP 7.4 – 8.3 · WooCommerce 5.0+ · WordPress Multisite


== Installation ==

1. Загрузите папку `wp-authorization-ru` в директорию `/wp-content/plugins/`
2. Активируйте плагин через меню «Плагины» в WordPress
3. Перейдите в **Настройки → Авторизация RU**
4. Зарегистрируйте приложение у нужного провайдера и получите Client ID и Client Secret
5. Скопируйте Redirect URI из страницы настроек плагина и вставьте в настройки приложения у провайдера
6. Введите Client ID и Client Secret, включите нужный провайдер и сохраните

== Получение ключей у каждого провайдера ==

**Яндекс ID:**
1. Откройте https://oauth.yandex.ru/ → «Зарегистрировать приложение»
2. Тип платформы: «Веб-сервисы»
3. Права доступа: login:info, login:email
4. Redirect URI: скопируйте из настроек плагина

**Mail.ru:**
1. Откройте https://o2.mail.ru/app/ → «Добавить приложение»
2. Тип: «Сайт»
3. Redirect URI: скопируйте из настроек плагина

**ВКонтакте:**
1. Откройте https://dev.vk.com/ → «Мои приложения» → «Создать»
2. Тип: «Веб-сайт»
3. В настройках → Безопасность → включите доступ к email
4. Redirect URI: скопируйте из настроек плагина

**Rambler:**
1. Откройте https://id.rambler.ru/apps → «Добавить приложение»
2. Тип: «Веб-сайт»
3. Redirect URI: скопируйте из настроек плагина

**MAX Мессенджер:**
1. Перейдите на https://max.ru/partner → Чат-боты → ваш бот → «Интеграция»
2. Получите Bot Token
3. В разделе «Мини-приложение» укажите URL вашего сайта
4. В настройках плагина введите Bot Token и @username бота

**Госуслуги (ЕСИА):**
* Только для юридических лиц (ИП, ООО) с УКЭП
* Зарегистрируйте систему на https://esia.gosuslugi.ru/console/tech/
* Получите mnemonic (Client ID) и УКЭП организации
* Для тестирования включите тестовую среду в настройках

== Frequently Asked Questions ==

= Где получить ключи для Яндекс и Mail.ru? =

Яндекс: https://oauth.yandex.ru/. Mail.ru: https://o2.mail.ru/app/. Подробные инструкции на странице настроек плагина.

= Где получить ключи для ВКонтакте? =

Зарегистрируйте приложение на https://dev.vk.com/ → «Мои приложения» → «Создать». Тип: «Веб-сайт». В настройках приложения включите доступ к email.

= Почему кнопка MAX выглядит иначе, чем другие? =

MAX не поддерживает стандартный OAuth 2.0 — вход происходит через Mini App. Кнопка ведёт на страницу бота. Когда пользователь открывает сайт через бота MAX, JavaScript автоматически определяет initData и выполняет вход через REST API.

= Требуется ли УКЭП для Госуслуг? =

Да, для продуктивной среды обязательно. Для тестирования включите тестовую среду в настройках плагина. Интеграция с ЕСИА доступна только юридическим лицам.

= Как работает автообновление с GitHub? =

Плагин раз в 12 часов запрашивает GitHub API (releases/latest). Если тег новее текущей версии — WordPress показывает стандартное уведомление об обновлении. Установка одним кликом. Папка плагина при этом переименовывается автоматически.

= Почему Яндекс пишет «Неверное значение поля»? =

Убедитесь, что вводите только один URL в поле. Для Яндекса — только /yandex/callback, для Mail.ru — отдельное приложение с /mailru/callback, и т.д.

= Что происходит, если пользователь уже зарегистрирован с таким email? =

Плагин найдёт существующий аккаунт по email и привяжет к нему OAuth ID провайдера. Все данные пользователя сохраняются.

= Что удаляется при удалении плагина? =

Настройки плагина, OAuth-идентификаторы пользователей и кэш обновлений. Сами аккаунты пользователей НЕ удаляются.

= Нужен ли REST API WordPress? =

Да. REST API должен быть доступен (не заблокирован плагинами безопасности).
