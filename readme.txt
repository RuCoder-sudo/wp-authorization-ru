Авторизация и регистрация через российские сервисы: Яндекс ID и Mail.ru OAuth. Поддержка WordPress и WooCommerce.

== Description ==

WP Authorization RU — плагин для авторизации пользователей WordPress через российские OAuth-провайдеры: **Яндекс ID** и **Mail.ru**. Версия 1.0.0.

**Два провайдера в одном плагине:**

* Яндекс ID — вход через аккаунт Яндекс (Яндекс Почта, Яндекс Диск и другие сервисы).
* Mail.ru — вход через аккаунт Mail.ru.

**Где появляются кнопки входа:**

* Стандартная страница входа WordPress (wp-login.php)
* Стандартная страница регистрации WordPress
* Страница «Мой аккаунт» WooCommerce (форма входа)
* Страница «Мой аккаунт» WooCommerce (форма регистрации)
* Страница оформления заказа WooCommerce — баннер «Быстрый вход перед оформлением»

**Что происходит при входе:**

* Пользователь нажимает кнопку «Войти через Яндекс» или «Войти через Mail.ru».
* Перенаправляется на сервис для подтверждения.
* После подтверждения возвращается на сайт уже авторизованным.
* При первом входе автоматически создаётся аккаунт WordPress с ролью «Подписчик».
* При повторном входе — распознавание по OAuth ID или email.
* Имя и фамилия из профиля провайдера сохраняются в профиле WordPress.

**Безопасность:**

* CSRF-защита через state-параметр (WordPress nonce + transient, срок действия 10 минут).
* Callback через WordPress REST API — чистый URL без query-параметров.
* Все входные данные проходят санитизацию.
* Редирект только на страницы того же сайта.

**Совместимость:** WordPress 5.8 – 6.7 · PHP 7.4 – 8.3 · WooCommerce 5.0+


== Installation ==

1. Загрузите папку `wp-authorization-ru` в директорию `/wp-content/plugins/`
2. Активируйте плагин через меню «Плагины» в WordPress
3. Перейдите в **Настройки → Авторизация RU**
4. Зарегистрируйте приложение у нужного провайдера и получите Client ID и Client Secret
5. Скопируйте Redirect URI из страницы настроек плагина и вставьте в настройки приложения у провайдера
6. Введите Client ID и Client Secret, включите нужный провайдер и сохраните

== Получение Client ID и Client Secret ==

**Яндекс ID:**

1. Откройте https://oauth.yandex.ru/ → «Зарегистрировать приложение»
2. Тип платформы: «Веб-сервисы»
3. Права доступа: login:info, login:email
4. Redirect URI: скопируйте из страницы настроек плагина

**Mail.ru:**

1. Откройте https://o2.mail.ru/app/ → «Добавить приложение»
2. Тип: «Сайт»
3. Redirect URI: скопируйте из страницы настроек плагина

== Frequently Asked Questions ==

= Где получить Client ID и Client Secret для Яндекс? =

Зарегистрируйте приложение на https://oauth.yandex.ru/. Тип платформы — «Веб-сервисы». Права: login:info и login:email. Redirect URI укажите тот, что показан в настройках плагина.

= Где получить Client ID и Client Secret для Mail.ru? =

Зарегистрируйте приложение на https://o2.mail.ru/app/. Тип — «Сайт». Redirect URI укажите тот, что показан в настройках плагина.

= Почему Яндекс пишет «Неверное значение поля» при вводе Redirect URI? =

Убедитесь, что вводите только один URL (без пробелов и без второго адреса). Для Яндекса используйте только URL с /yandex/callback, для Mail.ru — отдельное приложение с /mailru/callback.

= Что происходит, если пользователь уже зарегистрирован на сайте с таким email? =

Плагин найдёт существующий аккаунт по email и привяжет к нему OAuth ID провайдера. Пользователь войдёт в свой обычный аккаунт.

= Можно ли включить одновременно Яндекс и Mail.ru? =

Да. Оба провайдера работают независимо. Если оба включены — на странице входа появятся две кнопки.

= Поддерживается ли WooCommerce? =

Да. Кнопки входа появляются на странице «Мой аккаунт» (формы входа и регистрации) и на странице оформления заказа.

= Нужен ли REST API WordPress? =

Да. Плагин использует WordPress REST API для обработки callback от OAuth-провайдеров. REST API должен быть доступен (не заблокирован плагинами безопасности).
