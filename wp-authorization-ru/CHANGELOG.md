## [1.0.3] - 2026-06-10

### Added
- **Журнал событий (`WP_Auth_Ru_Logger`)** — новый класс `class-wp-auth-ru-logger.php`.
  - Хранит последние 200 записей в `wp_options` (без autoload, не нагружает каждую страницу).
  - Уровни: `INFO`, `SUCCESS`, `WARNING`, `ERROR`.
  - Методы: `log()`, `get_logs()`, `clear_logs()`, `get_stats()`, `mask_email()`.
  - Email-адреса в журнале автоматически маскируются: `ivan@ya.ru` → `i***@ya.ru`.
- **Вкладка «📋 Журнал событий»** на странице настроек плагина.
  - Цветная статистика: Всего / Успех / Ошибки / Предупреждения / Инфо.
  - Разбивка по провайдерам: сколько всего входов/регистраций/ошибок.
  - Таблица событий с цветовым кодированием строк по уровню.
  - Кнопка «🗑️ Очистить журнал» с подтверждением.
  - Кнопка «🔄 Обновить» для получения свежих данных.
  - Красный счётчик ошибок в заголовке вкладки.
- **Полное логирование во всех 6 провайдерах** — каждый ключевой шаг записывается в журнал:
  - `INFO`: начало OAuth flow (IP-адрес), получение профиля.
  - `SUCCESS`: токен получен, вход существующего пользователя, регистрация нового пользователя (с маскированным email).
  - `WARNING`: недействительный state (CSRF-атака), просроченный initData (MAX).
  - `ERROR`: ошибка запроса токена, ошибка получения профиля, ошибка создания пользователя, ошибка провайдера.
- **Логирование блокировок** в классе `WP_Auth_Ru_Compliance`:
  - `WARNING` при попытке входа через иностранный сервис (`check_foreign_user`, `wp_login`).
  - `WARNING` при блокировке регистрации через иностранный сервис.
- `uninstall.php` — добавлена очистка опции `wp_auth_ru_log` при удалении плагина.

### Changed
- Описание плагина (Plugin Name header) обновлено: явно указан журнал событий.
- Версия обновлена до `1.0.3`.

## [1.0.2] - 2026-06-10

### Added
- **Класс соответствия (`WP_Auth_Ru_Compliance`)** — защита от иностранных OAuth-сервисов согласно КоАП РФ.
- **Обнаружение иностранных плагинов** — автоматически выявляет 18+ известных плагинов (Nextend Social Login, Google Login miniOrange, WPBrigade, WooCommerce Social Login, Super Socializer и др.) и выводит красный баннер в панели администратора с требованием деактивировать их.
- **«Режим соответствия»** — переключатель в настройках. При включении: блокирует вход и регистрацию через иностранные OAuth-сервисы; пользователи с user_meta от иностранных плагинов получают ошибку и не могут войти; на странице wp-login.php выводится понятное объяснение.
- **Информационный блок о законе** (КоАП РФ) на странице настроек плагина — таблица штрафов: граждане 10–20 тыс. руб., должностные лица 30–50 тыс. руб., юридические лица 500–700 тыс. руб. Принят во 2 и 3 чтениях (законопроект А. Горелкина).
- **Цветной блок предупреждения** на странице настроек при обнаружении активных иностранных плагинов — со ссылкой на список плагинов.

### Changed
- Описание плагина (Plugin Name header) обновлено: явно указана блокировка иностранных сервисов.
- Версия обновлена до `1.0.2`.

## [1.0.1] - 2026-06-10

### Added
- **Автообновление с GitHub** — новый класс `WP_Auth_Ru_Updater`. WordPress проверяет последний релиз на GitHub каждые 12 часов. При появлении новой версии в панели управления появляется стандартное уведомление «Доступно обновление» — обновление устанавливается одним кликом без посещения WordPress.org. После установки папка плагина автоматически переименовывается и плагин реактивируется.
- **Авторизация через ВКонтакте (VK OAuth 2.0)** — кнопка «Войти через ВКонтакте» на всех страницах входа. Email возвращается напрямую в ответе token endpoint (особенность VK API). Иконка: `assets/Image/vk-svgrepo-com.svg`.
- **Авторизация через Rambler ID** — кнопка «Войти через Rambler». Используется OpenID Connect / OAuth 2.0 через `id.rambler.ru`. Scope: `openid email profile`. Иконка: `assets/Image/Rambler.svg`.
- **Авторизация через MAX Мессенджер (Mini App)** — кнопка «Войти через MAX». MAX не поддерживает стандартный OAuth 2.0; реализована авторизация через HMAC-SHA256 + initData (Bot Token, без Client ID/Secret). JavaScript на странице автоматически определяет `window.MaxApp.initData` и выполняет вход при открытии сайта из MAX. Иконка: `assets/Image/MAX.svg`.
- **Авторизация через Госуслуги (ЕСИА)** — кнопка «Войти через Госуслуги». Реализован стандартный OAuth 2.0 flow (ESIA); поддерживается тестовая среда `esia-portal1.test.gosuslugi.ru`. Продуктивная среда требует УКЭП юридического лица. Иконка: `assets/Image/gosusligi-logo.svg`.
- **`uninstall.php`** — при удалении плагина автоматически очищаются: настройки, transient'ы кэша, user_meta всех провайдеров (OAuth ID, маркер провайдера, отчество Госуслуг). Поддержка WordPress Multisite — очищает каждый подсайт сети.
- **`CHANGELOG.md`** — файл истории изменений в формате Keep a Changelog.

### Changed
- Страница настроек **Настройки → Авторизация RU** обновлена: добавлены отдельные блоки с инструкциями для всех 6 провайдеров, кнопки копирования Redirect URI, описание особенностей MAX и ЕСИА.
- **`readme.txt`** обновлён: добавлены все 6 провайдеров, расширен раздел FAQ, обновлен Changelog.
- **`README.md`** обновлён: сводная таблица провайдеров с требованиями, инструкции по настройке каждого.
- Версия плагина обновлена до `1.0.1` во всех файлах.

## [1.0.0]

### Added
- Первый релиз плагина.
- **Авторизация через Яндекс ID** (OAuth 2.0, `oauth.yandex.ru`). Права: `login:info`, `login:email`.
- **Авторизация через Mail.ru** (OAuth 2.0, `oauth.mail.ru`). Данные через `userinfo` endpoint.
- Кнопки входа на стандартных страницах WordPress: `wp-login.php` (вход и регистрация).
- Кнопки входа на страницах WooCommerce: «Мой аккаунт» (формы входа и регистрации), оформление заказа — баннер «Войдите перед оформлением».
- Автоматическое создание аккаунта WordPress при первом OAuth-входе (роль: Подписчик).
- Привязка к существующему аккаунту по email при совпадении.
- CSRF-защита через state-параметр (WordPress nonce + transient, срок жизни 10 минут).
- Callback через WordPress REST API — чистые URL без query-параметров (`/wp-json/wp-auth-ru/v1/{provider}/callback`).
- Страница настроек в **Настройки → Авторизация RU** с пошаговой инструкцией и кнопками копирования Redirect URI.
- WooCommerce HPOS (High-Performance Order Storage) совместимость.
- `readme.txt` в формате WordPress.org.
- `README.md` в формате GitHub.
